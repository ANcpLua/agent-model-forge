# Claude Workspace Memory

## Purpose

This workspace is optimized for a Claude-led development loop with explicit skills, explicit rules, and lightweight command briefs.

## Defaults

- map scope before changing code
- prefer evidence before recommendation
- stop when input assumptions are no longer defensible
- produce output in named sections when a skill contract asks for it

## Runtime posture

- local skills are the main unit of reusable behavior
- rules define hard boundaries
- agents define default review and execution posture
